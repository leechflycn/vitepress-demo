# 给静态博客加上灵魂：PVE 旁路部署 Twikoo 评论系统实战

当你搭好了一个极简的 VitePress 博客，有了独立域名，也写了几篇文章后，你会发现它少了点“人情味”——读者看完了，想吐槽一句，却发现连个评论框都没有。

纯静态博客天然不支持动态交互，但借助像 Twikoo 这样的无后端评论系统，我们完全可以把这块“动态拼图”补上。

## 架构解耦：为什么不在博客宿主机混装？

最初我想图省事，直接在跑着博客源码的虚拟机里（103）用 Docker 跑一个 Twikoo 后端。但后来采纳了老冯的建议：**架构必须解耦**。

博客宿主机只管前端源码和编译；任何对外的、涉及到数据库读写的后端服务，都应该切分出来。

于是，我直接在 PVE 宿主机上，一键秒开了一个**专属的 LXC 容器（编号 104，Debian 12系统）**，专门用来跑 Twikoo 后端。哪怕以后评论系统被 D 死了，也绝对不会影响博客主站的稳定。

## 踩坑录：那些环境与网络的折磨

在 LXC 里跑 Docker 安装 Twikoo 时，遇到了经典的双重网络打击。

### 1. DNS 罢工
LXC 刚建好，执行 `apt update` 就全部报错解析失败。排查发现是底层配置没把 DNS 传进来。
**解法**：直接暴力修改底层的 `/etc/resolv.conf`，强行注入了公共 DNS（如 `223.5.5.5`），网络瞬间打通。

### 2. Docker 镜像墙
国内直连 Docker Hub 的墙越来越高，拉取 Twikoo 的官方镜像一直卡在 `waiting` 直到超时。
**解法**：去配了国内的 Docker 镜像加速源（如各大云厂商的镜像库或者代理库），重启 Docker 后，秒级拉取成功并启动。

## 前端改造与 RSS 订阅附赠

Twikoo 后端（`192.168.1.104:8080`）跑起来后，剩下的就是前端的活儿了。

在 103 虚拟机的 `vitepress-demo` 源码中：
1. **注入组件**：我在全局的主题配置里，把 Twikoo 的前端组件给挂载进去了。现在每篇文章底下，都会自动浮现出清爽的评论框。
2. **顺手加上了 RSS**：既然动了配置，干脆一步到位。引入了 `feed` 模块，配置了一下 `config.js`。现在每次触发 `docs:build` 时，系统不仅会渲染网页，还会自动生成一个 `feed.xml` 放在根目录。那些喜欢用 RSS 阅读器的大佬，终于可以订阅我的博客了。

最后，执行了一把 `git commit` 和 `push`。眼看着 Cloudflare Pages 的流水线跑完，绿灯亮起。去线上刷一下，评论框和 RSS 订阅完美上线！
