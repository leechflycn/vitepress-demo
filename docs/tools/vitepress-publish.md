# 📝 小白实操：如何给自己的独立博客（VitePress）发布新文章？

当你成功搭建了像我一样极简清爽的独立博客（基于 VitePress + GitHub + Cloudflare Pages），肯定会面临一个问题：“我自己平时怎么发新文章呢？”

其实非常简单，不用写复杂的代码，你只需要像写日记一样写 Markdown 文件（即 `.md` 文件）就可以了！因为你的网站已经实现了**全自动部署流水线**，只要你提交文件，大概一分钟左右，新文章就会自动出现在网站上！

下面就是极简的发帖教程，小白一学就会！

---

## 📌 第一步：准备好你的文章和配图

首先，你要在本地电脑上用任何你喜欢的文本编辑器（比如系统自带的记事本、VS Code 或者 Typora 等 markdown 编辑器），把你想写的内容准备好。

- **文章格式**：必须是 Markdown 格式，也就是文件后缀名是 `.md`。例如：`my-new-post.md`。
- **图片要求**：如果文章里有配图，把图片准备好（推荐用英文命名，比如 `pic1.png`）。

## 📌 第二步：把图片放进 `public` 文件夹（可选）

如果你的新文章里带了图片，你需要先把图片放入博客代码库的一个固定地方。

1. 登录到你存放博客源码的地方（比如你虚拟机里的 `/home/leechfly/vitepress-demo` 目录）。
2. 找到 `docs/public` 文件夹。
3. 把你的图片（比如 `pic1.png`）传到这个文件夹里。
4. 在你的文章中引用这张图片时，只需写一行代码：`![这是图片说明](/pic1.png)`。（注意：路径前面一定要有 `/`）。

## 📌 第三步：把你的文章放到对应的分类目录下

接下来，把你的 `.md` 文章放到你想要显示的分类文件夹里。

比如，你写的是一篇关于“建站经验”的教程，你可以把它放在 `docs/tools/` 文件夹下。如果是普通的图文教程，你可以放在 `docs/tutorials/` 文件夹下。

- 举例：把你写好的 `my-new-post.md` 放进 `docs/tutorials/` 目录中。

## 📌 第四步：修改配置文件（让左侧菜单显示你的文章）

这一步很关键！文章放进去后，如果不在配置文件里“打个招呼”，网站的侧边栏菜单是不会显示它的。

1. 找到 `docs/.vitepress/config.js`（或 `config.mts` / `config.ts`）。
2. 打开它，找到你想放进去的 `sidebar`（侧边栏）位置。
3. 仿照之前的格式，在 `items` 数组里增加一行配置：
   ```javascript
   { text: "我是新文章的标题", link: "/tutorials/my-new-post" }
   ```
   *注意：这里不需要写 `.md` 后缀。*
4. 保存修改后的配置文件。

## 📌 第五步：使用 Git 推送，全自动发布！

所有的改动都做完后，接下来就是见证奇迹的时刻！

在终端（命令行）进入你博客根目录（`vitepress-demo`），执行这三个经典的 Git 命令：

```bash
# 1. 告诉系统把所有的改动添加进来
git add .

# 2. 给这次发帖写个备注
git commit -m "发布了一篇新文章"

# 3. 将改动推送到 GitHub 的云端！
git push origin main
```

**✅ 大功告成！**
只要这三行命令执行成功，一切就交给 Cloudflare 去全自动处理了。去泡杯咖啡，大约一两分钟后，刷新你的网站网址（比如 `https://你的域名`），你的最新大作就已经成功上线啦！是不是超简单？
