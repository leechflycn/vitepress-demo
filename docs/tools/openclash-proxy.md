# 旁路由 OpenClash 透明代理配置避坑

## 网络拓扑前提
如果你希望局域网内的设备（如 Windows 主控机、虚拟机）**不开启任何代理软件就能直接翻墙（透明代理）**，需要满足以下条件。

## 核心配置步骤

1. **指向网关和 DNS**：
   将需要透明代理的设备的 **默认网关** 和 **首选 DNS** 均手动指向你的旁路由 IP（例如：`192.168.1.2`）。
   > Linux (如 Ubuntu 虚拟机) 同样需要在 Netplan 或 NetworkManager 中修改。如果不改，AI 或部分命令行工具会因无法连通外网而报错。

2. **OpenClash 设置**：
   - 必须处于 **Fake-IP 模式**。
   - 旁路由的防火墙（lan 区域）必须开启 **IP 动态伪装 (MASQUERADE)**，否则流量无法正常转发。

3. **HTTP 407 认证拦截（关键大坑）**：
   如果你的 OpenClash 开启了“局域网共享”，**必须确保关闭“HTTP 代理端口认证”**（即不要设置账号密码）。否则局域网内的流量会被静默拦截，设备彻底断网且很难排查原因。

## 备选安全方案
如果你不想动全局网关和 DNS（怕把网络搞崩），最稳妥的排查方式是：在系统设置或浏览器中，手动将 HTTP/HTTPS 代理指向旁路由的 7890 端口（如 `192.168.1.2:7890`）。这样既能翻墙，又完全不影响局域网的内网通信。
